# Perforator / Ябко+ Tech Store
## Бізнес-документ

---

## 1. Назва системи та короткий опис

Perforator (Ябко+ Tech Store) — це навчальний вебзастосунок інтернет-магазину
техніки, який демонструє базові принципи побудови сучасних вебсистем.
Користувачі можуть переглядати каталог товарів, фільтрувати їх за брендами
та категоріями, переглядати детальну інформацію про товар і працювати з профілем.

Проєкт створений для освітніх цілей та не передбачає реальних платежів.
Архітектура побудована за клієнт-серверною моделлю з використанням FastAPI
на бекенді та React на фронтенді.

---

## 2. Користувачі (ролі)

- **Гість**
  - Перегляд каталогу товарів
  - Пошук та фільтрація
  - Перегляд сторінки товару

- **Зареєстрований користувач**
  - Усі можливості гостя
  - Реєстрація та вхід у систему
  - Перегляд профілю
  - Створення та перегляд замовлень (демо)

- **Адміністратор (майбутній)**
  - Керування товарами
  - Керування замовленнями
  - Керування користувачами

---

## 3. Основні сценарії використання

1. Користувач відкриває сайт і переглядає каталог товарів
2. Користувач застосовує фільтри та пошук
3. Користувач переглядає сторінку товару
4. Користувач реєструється або входить у систему
5. Авторизований користувач створює замовлення

---

## 4. Функціональні вимоги

### 4.1 Перегляд каталогу товарів
**Як гість або користувач** хочу переглядати список товарів,  
щоб ознайомитись з асортиментом магазину.

**AC:**  
- Given товари існують у базі  
- When GET /products  
- Then 200 OK + список товарів  

**API:**  
- GET /products → 200, 400

---

### 4.2 Фільтрація та пошук товарів
**Як гість або користувач** хочу фільтрувати товари,  
щоб швидко знайти потрібний товар.

**AC:**  
- Given query|brand|category, limit≤100, offset≥0  
- When GET /products?...  
- Then 200 OK + items, total  

**API:**  
- GET /products → 200, 400

---

### 4.3 Перегляд сторінки товару
**Як гість або користувач** хочу відкрити сторінку товару,  
щоб побачити детальну інформацію.

**AC:**  
- Given товар існує  
- When GET /products/{slug}  
- Then 200 OK  

**API:**  
- GET /products/{slug} → 200, 404

---

### 4.4 Реєстрація користувача
**Як гість** хочу зареєструватися,  
щоб отримати доступ до профілю.

**AC:**  
- Given username унікальний  
- When POST /auth/register  
- Then 201 CREATED + JWT  

**API:**  
- POST /auth/register → 201, 400, 409

---

### 4.5 Авторизація користувача
**Як користувач** хочу увійти в систему,  
щоб працювати зі своїм профілем.

**AC:**  
- Given валідні дані  
- When POST /auth/login  
- Then 200 OK + JWT  

**API:**  
- POST /auth/login → 200, 401

---

### 4.6 Створення замовлення
**Як авторизований користувач** хочу створити замовлення,  
щоб зафіксувати покупку.

**AC:**  
- Given JWT валідний  
- When POST /orders  
- Then 201 CREATED  

**API:**  
- POST /orders → 201, 400, 401

---

## 5. Нефункціональні вимоги (NFR)

- API p95 ≤ 200 мс при 50 RPS
- Частка помилок 5xx < 0.5% на добу
- Доступність SLO 99.9%/міс
- JWT + TLS ≥ 1.2
- Rate-limit: 10 req/min/IP на /auth/*
- API stateless, горизонтальне масштабування

---

## 6. Обмеження

- Відсутні реальні платежі
- Немає інтеграції з банківськими сервісами
- Відсутня повноцінна адмін-панель
- Проєкт не є комерційним
