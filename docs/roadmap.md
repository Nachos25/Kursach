# Roadmap проєкту Perforator / Ябко+ Tech Store

Цей roadmap описує функції, які команда планує реалізувати протягом найближчих 3–4 тижнів, разом з критеріями прийняття (DoD) та метриками.

---

## W3 — Каталог товарів (читання)

**Мета:** надати користувачам зручний каталог товарів з фільтрацією та пошуком.

- **Backend:**  
  - GET /products з фільтрами (category, brand, q)  
  - Підтримка пагінації (limit/offset)  
  - Оптимізація запитів (індекси для пошуку та фільтрів)

- **Frontend:**  
  - Сторінка каталогу, пошук, фільтри  
  - Порожні стани (якщо товарів немає)  

- **Критерії прийняття / DoD:**  
  - p95 ≤ 200 мс при 50 RPS  
  - Частка помилок 5xx < 0.5%  
  - Логування request_id  
  - 3 е2е тести на каталог

---

## W4 — Авторизація та безпека

**Мета:** забезпечити реєстрацію, логін та безпечний доступ користувачів.

- **Backend:**  
  - POST /auth/register → реєстрація з JWT  
  - POST /auth/login → логін  
  - GET /auth/me → профіль користувача  
  - Rate-limit /auth/*: 10 req/min/IP  
  - Базові OWASP-чеклисти

- **Frontend:**  
  - Сторінки реєстрації та логіну  
  - Обробка помилок 401/403  
  - Зберігання токена в localStorage або cookie

- **DoD:**  
  - 401/403 кейси протестовані  
  - Security checklist пройдено  
  - Unit-тести для auth логіки  
  - JWT перевіряється на всіх захищених endpoint'ах

---

## W5 — Замовлення (CRUD)

**Мета:** дати користувачам можливість створювати та переглядати замовлення.

- **Backend:**  
  - POST /orders → створення замовлення  
  - GET /orders → список своїх замовлень  
  - Валідації (користувач авторизований, дані валідні)  
  - Soft-delete для видалення замовлень  

- **Frontend:**  
  - Форма створення замовлення  
  - Список історії замовлень у профілі  
  - Валідація на клієнті

- **DoD:**  
  - Міграції БД застосовано  
  - Індекси під запити на orders  
  - Ідемпотентність POST /orders  
  - Unit-тести ≥90% на доменні класи

---

## W6 — Майбутні модулі / опціонально

- **Аналітика та нотифікації:**  
  - Лічильники переглядів/створень товарів  
  - Черга для email/SMS (скелет)  

- **Адмін-панель (скелет):**  
  - Додавання/редагування товарів  
  - Перегляд замовлень  

- **DoD:**  
  - Дашборд подій (metrics)  
  - Алерти на відставання черги  
  - Unit-тести та інтеграційні тести для нових модулів
