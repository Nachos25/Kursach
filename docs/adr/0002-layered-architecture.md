# ADR 0002 — Layered Architecture

## Context
Вибрано модульний моноліт. Потрібно визначити шари і правила залежностей.

## Layers
1. **api/** — контролери та роутери, приймають запити, перетворюють у DTO, повертають відповіді.
2. **service/** — бізнес-логіка, use-cases, валідація, мапінг DTO ↔ domain.
3. **domain/** — сутності, інваріанти, базові методи.

## Dependency Rules
- api → service → domain  
- Заборонено: api напряму у domain або БД  
- Заборонено: service напряму у api
