# ADR 0001 — Architecture Style

## Context
Проєкт Perforator зараз на етапі навчальної реалізації. Команда невелика, немає складних DevOps процесів, важлива швидка інтеграція.

## Decision
Обираємо **модульний моноліт** як базову архітектуру.  
- Всі модулі: Catalog, Users, Orders, Payments поки що в одному репозиторії.
- Кожен модуль логічно ізольований через папки і шари (api/service/domain).

## Alternatives
- Мікросервіси: поки що надмірно складно для маленької команди; можливо пізніше.

## Consequences
- Просте розгортання, швидкий старт.
- Легше тестувати та дебажити.
- Майбутній перехід на мікросервіси можливий при зростанні навантаження.
